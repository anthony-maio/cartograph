Cartograph Privacy Policy

Last updated: March 15, 2026

Cartograph is a local-first repo analysis tool for coding agents. By default, Cartograph runs on your machine and stores working artifacts in your local user cache.

What Cartograph collects

Cartograph does not send usage telemetry, analytics events, repository contents, or cached analysis artifacts to Anthony Maio or Making Minds by default.

Cartograph may read local repositories, local files, or GitHub repository URLs only when you explicitly run a Cartograph command or invoke it through a supported agent host such as Claude Code, OpenClaw, or MCP.

Where your data lives

By default, intermediate artifacts such as analysis results, task context, dependency maps, and generated summaries are stored in your local Cartograph user cache. Cartograph does not write those artifacts into your repo unless you explicitly export them to a path you choose.

Third-party services

Some Cartograph features can call third-party services if you configure them. For example, if you provide API keys for Gemini, OpenAI, or OpenRouter, Cartograph may send prompts and selected repository context to those providers in order to generate summaries or documentation.

If you analyze a GitHub URL, Cartograph may access GitHub-hosted repository content needed for that analysis. Your use of those services is also governed by the privacy policies and terms of the relevant provider.

Claude, OpenClaw, and MCP integrations

Cartograph includes install adapters, skills, commands, and optional MCP configuration for supported hosts. Those integrations run Cartograph locally unless you configure an external provider. Host applications may maintain their own logs, settings, or conversation history independent of Cartograph.

Your choices

You control when Cartograph runs, which repositories it reads, which files are exported, and whether any external AI provider or remote repository source is used.
